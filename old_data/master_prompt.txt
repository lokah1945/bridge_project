0.1 Aturan Anti-Halusinasi
- Jangan pernah menyatakan "selesai", "100% berhasil", "production ready",
  "sudah berjalan", atau sejenisnya TANPA bukti eksekusi nyata (log, output

- Format pelaporan untuk setiap klaim penting:
    CLAIM   : <pernyataan>
    EVIDENCE: <log / output / isi file>
    VERDICT : PASS / FAIL / BLOCKED

- Jika tidak yakin -> tulis "UNVERIFIED". Jangan menebak jumlah model, jumlah

0.2 Loop Eksekusi
  BUILD -> RUN -> OBSERVE LOG -> FIX -> RE-RUN -> VERIFY -> ULANGI

    b) Ditemukan BLOCKER nyata (disertai error message asli + langkah yang

BAGIAN 1 — KONTEKS PROYEK
================================================================================

Bridge-Project adalah gateway otomatisasi yang mengubah Web UI dari beberapa
provider AI (Arena.ai, Qwen/chat.qwen.ai, DeepSeek/chat.deepseek.com) menjadi
SATU API lokal yang kompatibel dengan format OpenAI
(/v1/models, /v1/chat/completions), sehingga bisa dipakai oleh aplikasi lain
di jaringan lokal (Open WebUI, LibreChat, Cursor, dsb) seolah-olah memanggil
endpoint OpenAI biasa.

Arsitektur Hub-and-Spoke terdiri dari 2 komponen:

  1. bridge-server  -> "Session Hub". Windows, Node.js, browser headfull yang
     sudah login manual ke akun (Google, Arena, Qwen). SUDAH BERJALAN & SUDAH
     LOGIN. Lihat Bagian 3.

  2. bridge-client  -> "Automation Engine". Linux, Python/FastAPI. INI YANG
     AKAN DIBANGUN ULANG SECARA MENYELURUH di prompt ini.

BAGIAN 2 — TUJUAN AKHIR (PRIMARY OBJECTIVE)
================================================================================

Bangun ulang bridge-client (Python) dari nol, secara end-to-end, sehingga:

  1. Mengambil session (cookies, user-agent, headers) dari bridge-server yang
     sudah aktif, mengenkripsinya (AES-256/Fernet), dan menyimpannya lokal.
  2. Menggunakan session tersebut untuk menjalankan browser Playwright HEADLESS
     miliknya sendiri, lalu MELAKUKAN OTOMASI DOM langsung ke arena.ai,
     chat.qwen.ai, dan chat.deepseek.com (client = satu-satunya eksekutor).
  3. Melakukan DYNAMIC MODEL DISCOVERY nyata (bukan list statis/seed) untuk
     ketiga provider, termasuk:
       - Arena: 4 varian modality URL (/text, /search, /image, /code /direct)
       - Qwen : daftar "Primary" + daftar "Expand More" (tersembunyi)
       - DeepSeek: dropdown model utama
     Hasil discovery disimpan di model.json, di-refresh otomatis setiap 60
     menit via background task, dan runtime /v1/models membaca dari file ini
     (instan, tidak scraping setiap request).
  4. Mengekspos API OpenAI-compatible: GET /v1/models, POST /v1/chat/completions
     (mendukung field extra_params untuk kontrol fitur spesifik provider — lihat
     Bagian 15).
  5. 100% stateless / zero footprint: tidak meninggalkan jejak history di akun
     provider (gunakan endpoint anonim/temporary + clear cookies setelah tiap
     eksekusi).
  6. Tervalidasi dengan test_all_models.py yang mengirim prompt "hallo" ke
     SETIAP model yang TERDETEKSI SECARA NYATA dan melaporkan matriks
     PASS/FAIL beserta isi balasan.

BAGIAN 3 — STATUS SAAT INI: BRIDGE-SERVER (SUDAH JALAN — JANGAN DIBANGUN ULANG)
================================================================================

bridge-server SUDAH BERJALAN dan TIDAK PERLU dibangun ulang. Perlakukan sebagai
black-box service yang sudah benar. Jangan mengubah kode server, jangan
menambahkan endpoint eksekusi (/proxy-chat atau sejenis) di sisi server — lihat
larangan di Bagian 4 & 18.

3.1 Spesifikasi server (referensi, untuk dipahami kontraknya saja)
  - Platform : Windows Server, headfull Chrome.
  - Stack    : Node.js + Express + playwright-extra + puppeteer-extra-plugin-
               stealth + manual CDP overrides (navigator.webdriver, languages,
               window.chrome, sec-ch-ua headers) + chromium.launchPersistentContext
               (session disimpan permanen per-profile di disk).
  - Multi-profile: saat dijalankan (npm start) ada menu CLI:
       (1) Start Server (pilih profile jika >1)
       (2) Add New Profile
       (3) Delete Saved Profile
  - Login manual dilakukan via endpoint /open (membuka URL di browser headfull
    persistent agar user bisa login manual / selesaikan MFA / captcha).

3.2 Endpoint yang TERSEDIA dan SUDAH AKTIF
  - BRIDGE_SERVER_URL = http://host.zerotier.my.id:9877   (tunnel ZeroTier,
    SUDAH AKTIF, SUDAH LOGIN ke Google, arena.ai, dan chat.qwen.ai)
  - GET  /health
        -> {"status": "online"}
  - GET  /open?url=<url>
        -> membuka <url> di browser headfull persistent (untuk login manual)
  - GET  /get-session/:provider     (provider ∈ {arena, qwen, deepseek})
        -> {
             "cookies": [ ...array cookie object Playwright (name, value,
                          domain, path, expires, httpOnly, secure, sameSite)... ],
             "user_agent": "Mozilla/5.0 (Windows NT 10.0; Win64; x64) ...",
             "headers": { "Accept-Language": "en-US,en;q=0.9", ... }
           }
  - (Opsional, jika versi server di repo sudah memiliki ini)
    GET /scan-models?provider=<arena|qwen|deepseek>
        -> hasil scraping model & fitur menggunakan browser server yang SUDAH
           login. Bridge-client BOLEH memakai endpoint ini sebagai CROSS-CHECK
           tambahan, TAPI mekanisme discovery UTAMA tetap dijalankan oleh
           client sendiri (lihat Bagian 4).

3.3 Statelessness DeepSeek
  Akun deepseek BELUM tentu sudah login di bridge-server pada saat ini
  (riwayat percakapan menyebut Arena & Qwen sudah login). Jika
  /get-session/deepseek mengembalikan cookies kosong, bridge-client harus tetap
  berjalan untuk arena & qwen, dan melaporkan status DeepSeek sebagai
  "NO_SESSION" di model.json (bukan error fatal yang mematikan seluruh server).

Ini adalah keputusan arsitektur TERAKHIR setelah beberapa kali bolak-balik di

Model yang BENAR ("Session-Provider / Executor-Client"):

  bridge-server (Windows)
      = Session Authority SAJA.
      - Tugas: login manual sekali, jaga browser headfull tetap hidup/login,
        sediakan cookies/headers via /get-session/:provider.
      - TIDAK menjalankan prompt apa pun. TIDAK ada logic chat di server.
      - Boleh dimatikan setelah client sinkron; hanya perlu dinyalakan lagi
        saat client butuh re-sync (session expired/revoked).

  bridge-client (Linux)  <-- INI YANG DIBANGUN
      = Satu-satunya eksekutor.
      - Sinkron sekali (atau saat expired) -> simpan terenkripsi lokal.
      - Menjalankan Playwright headless miliknya sendiri, inject cookies dari
        session yang disimpan, lalu OTOMASI DOM langsung ke
        arena.ai / chat.qwen.ai / chat.deepseek.com.
      - Expose API OpenAI-compatible ke jaringan lokal.

  Siklus hidup:
    [Server: login manual sekali] -> [Client: sync session -> simpan terenkripsi]
       -> [Client: operasi mandiri, server boleh OFF]
       -> [jika provider menolak session (401/403/redirect ke login)]
       -> [Client: re-sync ke server (server harus ON sebentar)] -> lanjut.

  Motto proyek: "Server login, client ambil, server boleh lupa (forget)."

BAGIAN 8 — providers/base.py (BaseProvider — Automation Core)
================================================================================

Tanggung jawab BaseProvider:
  1. Menyimpan session_data (cookies, user_agent, headers).
  2. Lazy-launch browser Playwright headless + context dengan cookies di-inject.
  3. Menerapkan STEALTH MANUAL via CDP (lihat larangan playwright-extra/
     playwright_stealth di Bagian 18 poin 1) — ini sudah pernah TERVALIDASI
     bekerja di iterasi terakhir histori chat.
  4. Method abstrak list_models(**kwargs) dan execute(model_id, prompt, params).
  5. cleanup() — WAJIB dipanggil setelah setiap request (di blok finally pada
     client.py) untuk menutup browser & menghindari memory/process leak.

Referensi implementasi (titik awal yang sudah tervalidasi — sesuaikan/perkuat
sesuai temuan discovery nyata, JANGAN dipersempit):

----------------------------------------------------------------------
from abc import ABC, abstractmethod
from typing import Any, Dict, List, Optional
from playwright.async_api import async_playwright, Page, BrowserContext

DEFAULT_UA = ("Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 "
               "(KHTML, like Gecko) Chrome/124.0.0.0 Safari/537.36")

class BaseProvider(ABC):
    def __init__(self, session_data: Dict[str, Any]):
        self.session_data = session_data or {}
        self.cookies = self.session_data.get("cookies", [])
        self.user_agent = self.session_data.get("user_agent", DEFAULT_UA)
        self.playwright = None
        self.browser = None
        self.context: Optional[BrowserContext] = None

    async def _setup_browser(self) -> Page:
        if not self.browser:
            self.playwright = await async_playwright().start()
            self.browser = await self.playwright.chromium.launch(
                headless=True,
                args=['--disable-blink-features=AutomationControlled']
            )
            self.context = await self.browser.new_context(user_agent=self.user_agent)
            if self.cookies:
                await self.context.add_cookies(self.cookies)

        page = await self.context.new_page()

        # Manual CDP Stealth (gantikan playwright-extra/playwright_stealth)
        cdp = await page.context.new_cdp_session(page)
        await cdp.send('Page.addScriptToEvaluateOnNewDocument', {
            'source': '''
                Object.defineProperty(navigator, 'webdriver', {get: () => undefined});
                Object.defineProperty(navigator, 'languages', {get: () => ['en-US','en']});
                Object.defineProperty(navigator, 'plugins', {get: () => [1,2,3]});
                Object.defineProperty(window, 'chrome', {get: () => ({runtime: {}})});
            '''
        })
        await cdp.send('Network.setExtraHTTPHeaders', {
            'headers': {
                'sec-ch-ua': '"Chromium";v="124", "Google Chrome";v="124", "Not-A.Brand";v="99"',
                'sec-ch-ua-mobile': '?0',
                'sec-ch-ua-platform': '"Windows"',
            }
        })
        return page

    @abstractmethod
    async def list_models(self, **kwargs) -> Any: ...

    @abstractmethod
    async def execute(self, model_id: str, prompt: str, params: Dict[str, Any]) -> str: ...

    async def cleanup(self):
        try:
            if self.browser:
                await self.browser.close()
            if self.playwright:
                await self.playwright.stop()
        except Exception:
            pass
----------------------------------------------------------------------

Catatan implementasi:
  - Perhatikan `page.context.new_cdp_session(page)` (bukan `page.context()` —
    di Python API, `context` adalah PROPERTY, bukan method seperti di Node.js;
    versi sebelumnya di histori chat salah menulis `page.context()` yang akan
    error di Python).
  - Jika butuh user-agent lain per provider, override via konstruktor /
    kwargs, jangan hardcode banyak versi base.py berbeda.

BAGIAN 14 — client.py (FastAPI Gateway)
================================================================================

14.1 SessionManager
  - Direktori sessions/, file <provider>.bin (JSON terenkripsi Fernet).
  - save_session(provider, data): tambahkan field "_saved_at" (ISO timestamp),
    encrypt, tulis ke file.
  - load_session(provider): decrypt, return dict atau None jika file tidak ada
    / gagal decrypt.
  - is_expired(session): bandingkan _saved_at + SESSION_TTL_HOURS dengan now().
  - get_effective_session(provider):
      1. session = load_session(provider)
      2. jika session is None atau is_expired(session):
           coba GET {BRIDGE_SERVER_URL}/get-session/{provider} (timeout wajar,
           misal 15s)
           - jika sukses (200) -> save_session(provider, response_json) ->
             gunakan ini
           - jika gagal (timeout/connection error/non-200):
               - jika session lokal lama MASIH ADA (walau "expired" by TTL)
                 -> log WARNING, tetap gunakan session lama (graceful
                    degradation / mode offline) sesuai prinsip "sync once,
                    use many"
               - jika TIDAK ADA session lokal sama sekali -> raise error
                 jelas: "AUTH_REQUIRED: bridge-server unreachable dan tidak
                 ada session lokal untuk provider '<provider>'"
      3. return session

  - Auto-refresh saat eksekusi gagal karena auth: jika provider.execute()
    melempar error yang mengindikasikan sesi tidak valid (redirect ke halaman
    login, 401/403, atau elemen halaman menunjukkan "Sign in"), maka:
      a. paksa re-sync (panggil get-session lagi ke BRIDGE_SERVER_URL, abaikan
         cache lokal),
      b. retry execute() SATU KALI dengan session baru,
      c. jika tetap gagal -> kembalikan error informatif ke caller (jangan
         retry tanpa batas).

14.2 Model Cache System (model.json)
  - Struktur file model.json (contoh skema):
      {
        "updated_at": "<ISO timestamp>",
        "providers": {
          "arena": {
            "status": "ok" | "error" | "stale",
            "last_error": null | "<pesan error>",
            "models": {
              "text":   ["<model>", ...],
              "search": ["<model>", ...],
              "image":  ["<model>", ...],
              "code":   ["<model>", ...]
            }
          },
          "qwen": {
            "status": "...",
            "last_error": null,
            "models": ["<model>", ...]
          },
          "deepseek": {
            "status": "...",
            "last_error": null,
            "models": ["<model>", ...]
          }
        }
      }

  - update_model_cache():
      Untuk setiap provider:
        - session = await get_effective_session(provider)
        - instance = ProviderClass(session)
        - try:
            - arena: jalankan list_models(modality) untuk ke-4 modality, isi
              dict per modality
            - qwen/deepseek: jalankan list_models()
            - status = "ok", last_error = None
          except Exception as e:
            - JANGAN timpa "models" yang sudah ada sebelumnya dengan data
              kosong/seed. Pertahankan data lama (jika ada), set
              status="stale"/"error", last_error=str(e).
        - finally: await instance.cleanup()
      Tulis seluruh model.json (atomic write: tulis ke file temp lalu rename).

  - Startup FastAPI:
      @app.on_event("startup")
      async def startup():
          await update_model_cache()              # WAJIB di-AWAIT, bukan
                                                    # fire-and-forget, agar
                                                    # model.json terisi NYATA
                                                    # sebelum request pertama
          asyncio.create_task(model_cache_loop())  # loop background

      async def model_cache_loop():
          while True:
              await asyncio.sleep(MODEL_CACHE_REFRESH_MINUTES * 60)
              await update_model_cache()

14.4 Endpoint POST /v1/chat/completions
  Request body:
      {
        "model": "bridge/<provider>/[<modality>/]<model_id>",
        "messages": [{"role": "user", "content": "..."}],
        "extra_params": { ... lihat Bagian 15 ... },
        "stream": false
      }

  Alur:
    1. parsed = registry.parse(model)  -> provider_name, model_id, (modality)
    2. session = await get_effective_session(provider_name)
    3. provider = ProviderClass(session)
    4. try:
         prompt = messages[-1]["content"]
         params = extra_params or {}
         if provider_name == "arena": params["modality"] = modality
         result_text = await provider.execute(model_id, prompt, params)
       except <auth-error>:
         ... (lihat 14.1 auto-refresh) ...
       finally:
         await provider.cleanup()
    5. Bentuk response OpenAI:
         non-stream:
           {
             "id": "chatcmpl-...",
             "object": "chat.completion",
             "model": "<model asli>",
             "choices": [{
               "index": 0,
               "message": {"role": "assistant", "content": result_text},
               "finish_reason": "stop"
             }]
           }
         stream=true:
           StreamingResponse yang mengirim result_text dalam beberapa chunk
           SSE bergaya OpenAI (`data: {...}\n\n` ... `data: [DONE]\n\n`).

14.5 Endpoint GET /health
  -> {"status": "ok", "model_cache_updated_at": "<dari model.json>"}
